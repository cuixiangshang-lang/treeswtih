## 三、图片文件清单

建议在 `images/` 文件夹中按以下结构存放：

```
images/
├── hero.jpg              # Hero 首屏大图（必填）
├── banner-1.jpg          # 全屏横幅 1（必填）
├── banner-2.jpg          # 全屏横幅 2（必填）
├── about.jpg             # About 右侧竖图（必填）
├── timeline/             # 时间线照片
│   ├── 2015-01.jpg
│   ├── 2015-02.jpg
│   └── ...
├── friends/              # 同行者头像
│   ├── liming.jpg
│   ├── zhangwei.jpg
│   └── ...
└── gallery/              # 画廊照片
    ├── 01.jpg
    ├── 02.jpg
    └── ...
```

---

## 四、修改流程

1. 准备好所有图片，按上方结构命名并放入 `images/` 文件夹
2. 打开 `index.html`
3. 搜索 `📷` 或 `placeholder`，逐项替换为真实内容
4. 搜索需要修改的文字段落，直接编辑
5. 保存文件
6. push 到 GitHub 仓库
7. Vercel 自动部署（约 30 秒生效）

## 五、图片规格建议

| 用途 | 建议尺寸 | 格式 |
|------|---------|------|
| Hero 大图 | 1920×1080 或 3840×2160（2x） | JPG / WebP |
| 全屏横幅 | 1920×800 | JPG / WebP |
| 时间线照片 | 800×600 或更大 | JPG |
| 同行者头像 | 200×200（正方形） | JPG |
| 画廊照片 | 800×800 或 800×600 | JPG |
| About 竖图 | 3:4 比例，如 600×800 | JPG |

**压缩建议：** 上传前用 TinyPNG（tinypng.com）压缩，减小文件体积，提升加载速度。
